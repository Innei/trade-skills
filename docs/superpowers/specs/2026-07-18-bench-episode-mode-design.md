# Bench v2：接续制交易题（episode 模式）设计草案

日期：2026-07-18
状态：草案，未排期。前置：v1 单发模式跑出第一份全量榜之后再实施。

## 1. 动机

v1 的题是单发的：一个时点、一次决策、回放定胜负。它测的是「单次判断质量」，测不了：

- **操作次数**——同一段趋势里模型会做几笔交易，是拿住一笔还是频繁进出
- **交易管理**——止损后敢不敢按新结构重进；到目标后会不会顺势再开
- **情绪化行为**——亏损后下一笔是否加大仓位/立刻反手（报复性交易），连胜后是否过度自信

这些恰恰是真人交易者和纪律系统的分水岭。

## 2. 核心机制：平仓点重入（不是逐日轮询）

一道 episode 题复用 v1 的题目文件（fixtures + 20 根回放 K 线），流程改为循环：

1. 第 1 轮与 v1 完全相同：模型看 cutoff 前的数据下单。
2. 回放推进到这笔交易**终结**（触止损 / 触目标 / 未成交超时 / 到 horizon）。
3. 若回放窗口还有 ≥3 根 K 线：把行情数据**推进到平仓那根 K 线**（fixtures 视图向前滚动，指标重算），连同**上一笔的完整结果**（方向、入场、出场、盈亏 R、终结原因）注入下一轮 prompt，再次要求决策（可观望=结束该题）。
4. 直到窗口耗尽或模型主动观望。每题通常 1~3 轮，成本约为单发的 2 倍（远低于 StockBench 式逐日轮询的 20 倍）。

关键设计约束：

- **虚拟时钟随轮次推进**——第 N 轮的「今天」= 上一笔平仓 K 线的日期；防泄漏不变量按轮重新执行（任何数据不得超过该轮时钟）。
- **上一笔结果注入是结构化的**，不是散文：`{ direction, entry, exit, r, reason }`，由 runner 生成，杜绝模型自己复述时篡改记忆。
- **run_code 沙箱的预载数据随轮滚动**，与 fetch_kline 视图一致。
- 观望在第 1 轮走 v1 观望通道；第 2 轮起观望 = 主动结束，不再有对错判定（离场是合法终局）。

## 3. 判分

- 判分单位从「题」变为「轮」（一轮 = 一笔完整交易生命周期），单轮规则完全复用 v1 回放规则。
- 题层汇总：**题内累计 R**（各轮得分求和）为该题主分；与买入持有同窗收益对照。
- 新增行为指标（episode 独有）：
  - **平均操作次数** / 每题轮数分布
  - **亏后行为**：上一轮亏损的条件下，下一轮的方向翻转率、观望率——报复性交易检测器
  - **赢后行为**：上一轮盈利的条件下的同向续做率——趋势持续性利用率
  - **过度交易罚没**：题内累计 R 为正但轮数 ≥4 的题目占比（同样的钱用更多笔数赚到 = 摩擦成本更高）
- 三基线适配：买入持有天然单轮；抛硬币每次平仓后再抛；永远观望只有第 1 轮。

## 4. 实施拆分（届时再细化）

1. runner：episode 循环器（复用 cell 执行器，外加轮间状态机与 fixtures 滚动视图）
2. mock：fixtures 按「虚拟今天」切片的视图工厂（generate 不用改，数据都在题文件里）
3. scorer：轮级判分 + 题级累计 + 行为指标
4. report：episode 专属榜单段落
5. 题库不变——同一份 v1 数据集可同时跑单发榜和接续榜，两榜互为对照

## 5. 开放问题（实施前需拍板）

- 第 N 轮是否允许模型看到自己第 1 轮的完整推理（还是只给结构化结果）？倾向只给结果——测的是纪律，不是上下文记忆。
- 未成交（no_fill）算不算消耗一轮？倾向算：挂单等待也占用了窗口时间。
- horizon 是否要为 episode 加长（20 根 → 40 根）？加长意味着回放数据翻倍，题库要重新生成，倾向 v2.1 再说。
