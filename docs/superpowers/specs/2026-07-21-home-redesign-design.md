# 首页重设计:统一骨架 + 大盘/板块/资金流/事件日历

日期:2026-07-21
状态:已确认(brainstorming 结论);同日晚间按实机截图反馈修订,见文末「v2 修订」——与正文冲突处以 v2 为准

## 背景与目标

现在的首页按时段切两种形态(盘中「看盘 + 持仓」、盘后「复盘」),有三个缺口:

1. 盘前没有专门形态——开盘前想知道「今天有什么事」(隔夜行情、财报、宏观数据)得自己到处翻。
2. 资金流依赖预先生成的 chart JSON,没跑过 capital-rotation 当天就是空白;而长桥 `capital <SYM> --flow` 本来就能实时给逐分钟净流入。
3. 没有大盘和板块层——只见个股不见轮动。

目标:一套三时段共用的布局骨架,数据焦点随时段切换,开盘前扫一眼就心里有数。

## 布局骨架(三时段共用)

```
┌─ 顶条:SPY / QQQ / .VIX 涨跌(带时段标签)+ 市场温度计 ─────────┐
├─ 主栏(约 2/3)────────────────────┬─ 右栏(约 1/3)──────────┤
│ ① 个股卡片网格(自选 + 持仓)      │ 事件流(今日 / 本周)     │
│ ② 板块热力图 + 一句话轮动读法      │ 持仓摘要                 │
│ ③ 个股资金流排序条                │                          │
└──────────────────────────────────┴──────────────────────────┘
```

- **顶条**:现有 QuoteBar 改造。指数参照固定打头(SPY / QQQ / .VIX),右侧市场温度计——长桥 `market-temp US` 的温度(0–100)渐变条 + 估值/情绪分项文字。
- **个股卡片**:现有 SymbolCard 扩展。新增当日净流入数字 + 迷你走势;临近财报(7 天内)打财报角标;持仓打持仓角标。
- **板块热力图**:一格一个板块 ETF,面积 = 当日成交体量,颜色 = 净流入方向与强度(绿进红出,7 档);底部图例 + 一句话轮动读法(如「钱在进半导体、出能源」,规则生成:最大流入板块 + 最大流出板块)。点击格子跳该 ETF 的 symbol 页。
- **个股资金流排序条**:自选 + 持仓全部个股按当日净流入排序的横向签名条。
- **事件流**:按时间列今日与本周事件。财报事件(橙色边)来自自选∪持仓逐股财报日历,标注属持仓还是自选;宏观事件(紫色边)来自宏观日历(CPI / FOMC / 非农等)。
- **持仓摘要**:总市值、当日/隔夜盈亏、仓位分布一行。

## 三时段换重点

| 区块 | 盘前(04:00–09:30 ET) | 盘中(09:30–16:00) | 盘后 / 休市 |
| --- | --- | --- | --- |
| 个股卡片 | 盘前价 + 隔夜盘涨跌 + 昨收 | 实时价 + AI 点评 + 预警角标(现看盘板信息) | 收盘定格,收窄成条 |
| 主栏主位 | 个股卡片 | 个股卡片 | 复盘板(RecapBoard)升主位 |
| 资金流(板块 + 个股) | 定格昨日全天 | 当日实时,60s 刷新 | 定格当日全天 |
| 事件流 | 今日事件置顶 | 已出数据打勾附实际值(如「CPI 2.9% = 预期」),未出显示倒计时 | 「盘后异动 → 明日事件」 |
| 右栏持仓 | 摘要 + 隔夜盈亏 | 升到右栏顶部,实时盈亏,近止损标 ⚠ | 摘要 + 当日盈亏 |

历史日期(`?date=` 回看)沿用盘后形态,数据来自落盘的 chart / recap,与现状一致。

## 内核侧改动

### 1. `board` WS 通道扩展

现有 `OverviewBoard` 每行(`OverviewRow`)新增:

- `net_inflow`:当日净流入(`getFlow(symbol)` 逐分钟序列求和),60s 缓存,与现有 60s chart 重建节奏对齐。
- 顶层新增 `market`:`{ temperature, valuation, sentiment, description }`(`market-temp`,10 分钟缓存)与 `sectors`:板块 ETF 数组 `{ symbol, label, net_inflow, turnover, pct }`。

板块 ETF 清单可配置,默认:SMH(半导体)、XLK(科技)、IGV(软件云)、XLF(金融)、XLE(能源)、XLV(医疗)。清单跟随用户设置存储,与自选列表同一套持久化。

### 2. 新接口 `overview.events`

- 输入:无(内部取自选 ∪ 持仓)。
- 行为:逐股 `getEarningsCalendar(symbol, today)` + `getMacroCalendar`,合并成按时间排序的事件数组 `{ ts, kind: 'earnings' | 'macro', symbol?, title, detail, owned }`。
- 缓存:按交易日缓存,盘前首次访问触发拉取;失败单条降级不整体报错。

盘中「已出数据打勾附实际值」依赖宏观日历返回的实际值字段;若长桥当日不回填实际值,则只展示「已发布」状态,不显示数值(实现时确认字段)。

## Web 侧改动

- `features/home/` 重构:`Home.tsx` 保持时段分支调度,新增 `IndexTempBar`(顶条)、`SectorHeatmap`、`FlowRank`(个股排序条)、`EventRail`,`SymbolCard` 扩展净流入与角标;`RecapBoard`、`PositionsCard` 复用。
- 单文件不超过 500 行、组件不超过 300 行,按现有 feature 目录习惯拆分。
- 热力图用 CSS grid 自绘(格子数 ≤ 8,不需要 treemap 算法库);面积映射用两档行高 + 列宽比例近似即可,不追求精确面积比。

## 错误处理

- 任何一路数据失败不拖垮整页:热力图、事件流、资金流各自显示重试提示,行情卡片照常。
- 沿用现有 `degraded` / `DataAgeBadge` 机制标注数据时效;资金流与温度数据带各自的定格时间戳。
- 长桥限流:板块 + 个股逐股拉 flow 的聚合在内核串行排队,复用现有 CLI 调用的节流通道。

## 测试

- 内核:净流入求和、板块聚合、事件合并排序、按日缓存与失效——单测。
- Web:三时段分支渲染(盘前/盘中/盘后各断言主位与右栏顺序)、热力图色档映射、事件流打勾逻辑——沿用现有组件测试模式。

## v2 修订(2026-07-21 晚,依据实机截图反馈)

第一版上线后实机太乱,根因是信息重复(净流入出现三处)和长尾等权平铺。修订如下:

### 主栏结构改为三段

1. **看盘卡**(不变):有 intraday 分析或持仓的票出大卡,净流入并入卡内。
2. **异动区**(替代自选小格墙):自选长尾只浮出「今天有事」的——|涨跌| ≥ 3%、或 4 天内有财报、或今日有预警;小格 = 代码 + 涨跌 + 净流入小字 + 财报角标。其余收成一行「+N 只平静」,点开才展开。
3. **市场全景 treemap**(替代板块 ETF 色块图):双 tab。
   - **默认 tab「自选+持仓」**:个股块按行业分组(截图式两层 treemap),面积 = 当日成交额(`quote.turnover`),颜色深浅 = 当日涨跌幅,持仓块加边框标识;行业归属用 core 内置的 symbol→行业静态映射表(现有自选预填,新自选无映射落「未分类」组)。
   - **tab「全市场」**:行业级全景,数据来自 `industry-rank --market US`(一次调用),块 = 行业,色 = 行业当日涨跌,块内标龙头股;面积按成员股数近似(行业无成交额数据)。10 分钟缓存。
   - 一句话轮动读法保留在图下,改为基于行业分组聚合生成。

### 删除项

- **个股资金流排序条删除**:净流入已在卡片与异动格上,且跨币种(7709.HK 港币)不可比,一根巨条压垮全图。
- **板块 ETF 六色块图删除**:被全景 treemap 取代;`sectors` 聚合与 SECTOR_ETFS 常量随之退役(内核字段可保留一版兼容,web 不再消费)。

### 数据面确认(2026-07-21 实查)

- `quote` 批量自带 `turnover`(当日成交额)与涨跌——个股块面积/颜色零额外调用。
- `industry-rank --market US` 给全部行业 + 当日涨跌 + 龙头股,一次调用。
- 个股→行业归属长桥拿不到(`company.sector` 恒为 0,`industry-peers` 不吐成员清单)——故用本地映射表。

### 其他修正(已随 v1 落地)

- QuickBar 快捷条回退为「有分析的 + 持仓」,不再铺全自选。
- 指数(`.` 前缀)与期权合约(`\d{6}[CP]\d+`)不进卡片/异动区/资金流聚合。

## 不做(YAGNI)

- 可拖拽 / 可配置 widget 仪表盘。
- 分组热力图(板块框内嵌个股)与 tab 切换——先上板块热力图,不够用再加。
- 板块级真实资金流(长桥无此 API,ETF 代理够用)。
- 新闻流、Trump 动态等其他信息源进首页。
